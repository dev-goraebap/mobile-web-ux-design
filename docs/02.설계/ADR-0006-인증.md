# ADR-0006 · 인증

> 상태: ✅ 결정됨
> 일자: 2026-06-22
> 관련: [프로젝트 브리프 §2](../01.기획/프로젝트-브리프.md), [ADR-0005](./ADR-0005-데이터-계층.md)

## 1. 맥락

회원 전용 기능(위시리스트·내 평점)은 인증이 필요하다. 하지만 이 프로젝트는 **백엔드가 없고 CSR 전용**이며, 목적은 인증 보안이 아니라 **반응형 UX**다. 회원가입·소셜 로그인·비밀번호 재설정은 범위 밖이고 **데모 계정 로그인**만 제공한다.

(자매 프로젝트 `angular-ssr-design`은 HttpOnly 쿠키 + SSR 쿠키 포워딩을 다루지만, 그건 SSR 인증 시연이 목적 — 우리는 채택하지 않는다.)

## 2. 검토한 대안

| 선택지 | 적합 | 비고 |
|--------|:---:|------|
| HttpOnly 쿠키 + 서버 검증 | ✕ | 백엔드 없음, SSR 주제 |
| **로컬 데모 계정 + 클라이언트 세션 토큰** ✅ | ◎ | 범위에 맞음 |

## 3. 결정

- **데모 계정**(`demo` / `demo1234`)을 클라이언트에서 검증한다(목 데이터).
- 로그인 성공 시 **세션 토큰을 localStorage에 저장**하고([ADR-0005](./ADR-0005-데이터-계층.md)), `AuthService`가 현재 사용자/세션을 시그널로 보관한다.
- 회원 라우트(`/wishlist`, `/me/ratings`)는 **`CanActivate` 가드**로 보호한다. 미인증 시 `/login`으로 보내고, **로그인 후 원래 페이지로 복귀**(returnUrl 쿼리 파라미터).
- 위시리스트·평점은 `userId`로 데이터와 연결한다([ADR-0005](./ADR-0005-데이터-계층.md)).
- 로그인 폼은 적응형 표현을 따른다([ADR-0002](./ADR-0002-적응형-표현-전략.md)).

## 4. 주의사항

- 클라이언트 전용 검증이라 **보안 목적이 아님**을 명확히 한다(데모 한정). 실제 인증은 범위 밖.
- 비로그인 상태에서 회원 액션(찜·평점) 시도 시 매끄럽게 로그인 유도 후 복귀하는 흐름을 UX로 설계한다.

## 5. 채택하지 않은 것

- **HttpOnly 쿠키 / SSR 포워딩**: 백엔드·SSR 전제(자매 프로젝트 주제).
- **실제 회원가입/소셜 로그인**: 범위 밖.

## 6. 결과

- 👍 서버 없이 회원/비회원 분기와 복귀 흐름을 시연.
- 👍 UX(로그인 유도→복귀)에 집중 가능.
- 👎 보안은 데모 수준 — 실서비스 전환 시 인증 계층 교체 필요(저장소 추상화로 완화).

## 7. 후속 작업

- [ ] `AuthService`(세션 시그널, login/logout)
- [ ] `CanActivate` 가드 + returnUrl 복귀
- [ ] 비로그인 회원 액션 → 로그인 유도 UX
- [ ] 로그인 페이지(적응형)
