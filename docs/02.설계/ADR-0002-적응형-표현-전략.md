# ADR-0002 · 적응형 표현 전략 (모달 ↔ 풀스크린/시트)

> 상태: ✅ 결정됨
> 일자: 2026-06-22
> 관련: [모바일 인터랙션 리서치 §2](../01.기획/모바일-인터랙션-리서치.md), [ADR-0001](./ADR-0001-ui-기술스택.md), [ADR-0003](./ADR-0003-뒤로가기-오버레이-닫기.md)

## 1. 맥락

같은 콘텐츠라도 환경에 맞는 표현이 다르다. 데스크톱에서는 영화 상세를 **모달**로 띄우는 게 자연스럽지만, 모바일의 작은 화면에 데스크톱식 모달을 띄우면 콘텐츠를 가리고 닫는 법이 모호해 **UX가 나쁘다**(NN/g, Apple HIG). 모바일에서는 **풀스크린 페이지**나 **바텀시트**가 맥락을 유지하며 편하다.

문제는 이걸 어떻게 **하나의 콘텐츠 컴포넌트**로 구현하면서 **딥링크·공유**를 깨뜨리지 않느냐다.

## 2. 검토한 대안

| 선택지 | 콘텐츠 중복 | 딥링크 | 모바일 적합 |
|--------|:---:|:---:|:---:|
| 데스크톱·모바일 컴포넌트 각각 작성 | ✕(중복) | △ | ○ |
| 항상 모달 | 없음 | △ | ✕(작은 화면에 모달) |
| 항상 풀스크린 페이지 | 없음 | ◎ | △(데스크톱은 과함) |
| **viewport 폭으로 표현만 스왑 + canonical URL** ✅ | 없음 | ◎ | ◎ |

## 3. 결정

**분기 기준은 디바이스가 아니라 viewport 폭**으로 하고(Material 3의 600dp 경계 참고), **콘텐츠는 한 번만 작성하고 표현 레이어만 스왑**한다.

화면별 매핑:

| 콘텐츠 | 데스크톱 | 모바일 | URL |
|--------|----------|--------|-----|
| **영화 상세** | 중앙 모달(scrim) | **풀스크린 페이지** | `/movies/:id` (canonical) |
| **평점 남기기/수정** | 모달 | **바텀시트** | (액션, route 불필요) |
| **검색 진입** | 드롭다운/인라인 | **풀스크린** | `/search?q=` |
| **필터** | 팝오버 | 바텀시트 | (액션) |

구현:
- CDK `BreakpointObserver`로 폭을 관찰하고, 공용 컴포넌트 **`AdaptiveDialog`**(데스크톱 모달 ↔ 모바일 풀스크린)와 **`AdaptiveSheet`**(데스크톱 모달/팝오버 ↔ 모바일 바텀시트)로 캡슐화한다([ADR-0001](./ADR-0001-ui-기술스택.md)).
- 콘텐츠 컴포넌트(`MovieDetail`, `RateMovieForm`)는 자신이 모달인지 페이지인지 **모르게** 작성한다. 표현은 래퍼가 결정한다.
- **canonical URL 원칙**: 공유 대상(영화 상세)은 항상 `/movies/:id`라는 진짜 라우트를 갖는다. 목록에서의 진입만 모달로 "가로채고", URL 직접 진입·새로고침 시에는 풀스크린 페이지로 렌더한다(Instagram/Pinterest 모델). 모달↔라우트 연결은 [ADR-0003](./ADR-0003-뒤로가기-오버레이-닫기.md)에서 다룬다.
- 초기 분기는 깜빡임을 줄이기 위해 **CSS 우선**(컨테이너/미디어 쿼리)으로 처리하고, JS 측정 기반 변경은 그다음.

full-screen 선택 신호(Material): 키보드 입력이 필요할 때, 변경이 즉시 저장되지 않을 때, 중첩 오버레이가 필요할 때.

## 4. 주의사항

- **시트를 여러 겹 쌓지 말 것**, 페이지 네비게이션을 시트로 흉내내지 말 것(NN/g). 다단계·복잡 작업은 풀스크린.
- 모바일 풀스크린/시트도 **명시적 닫기 버튼 + back으로 닫기**를 둘 다 제공한다([ADR-0003](./ADR-0003-뒤로가기-오버레이-닫기.md)).
- 태블릿 등 경계 폭에서의 분기 기준을 토큰(브레이크포인트)으로 한 곳에서 관리한다.

## 5. 채택하지 않은 것

- **데스크톱/모바일 컴포넌트 분리 작성**: 콘텐츠 로직이 중복되고 유지보수가 갈라진다.
- **항상 모달**: 모바일에서 작은 화면 차단·우발적 닫힘 문제.
- **항상 풀스크린**: 데스크톱에서 맥락 상실, 과한 전환.

## 6. 결과

- 👍 콘텐츠 1회 작성, 표현만 스왑 → 중복 없음.
- 👍 딥링크/공유/새로고침이 canonical URL로 안전.
- 👍 각 환경에 맞는 검증된 UX.
- 👎 `AdaptiveDialog`/`AdaptiveSheet` 래퍼와 라우트 가로채기 로직을 직접 구현해야 함.
