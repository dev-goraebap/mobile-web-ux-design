# ADR-0003 · 뒤로가기 & 오버레이 닫기

> 상태: ✅ 결정됨
> 일자: 2026-06-22
> 관련: [모바일 인터랙션 리서치 §1](../01.기획/모바일-인터랙션-리서치.md), [ADR-0002](./ADR-0002-적응형-표현-전략.md)

## 1. 맥락

모바일 사용자(특히 Android)는 오버레이(다이얼로그·바텀시트·모달)가 떠 있을 때 **뒤로가기를 누르면 페이지가 아니라 오버레이가 닫히길** 기대한다. 그러나:

- **CDK Overlay 자체엔 back-to-close가 없다**(라우트가 아니므로).
- 순수 `<dialog>`도 기본적으로 Android back으로 닫히지 않고 히스토리를 이동한다.
- 이 프로젝트는 **PWA**라 Ionic의 `ionBackButton` 방식은 동작하지 않는다.

따라서 오버레이 닫힘과 브라우저/OS 히스토리를 직접 연결하는 전략이 필요하다.

## 2. 검토한 대안

| 선택지 | back으로 닫힘 | 공유/딥링크 | 비고 |
|--------|:---:|:---:|------|
| 아무 처리 안 함 (버튼만) | ✕ | ✕ | 모바일 기대 위반 |
| `history.pushState` + `popstate` | ◎ | △ | 널리 쓰임. 직접 닫기 시 정리 함정 |
| `CloseWatcher` API | ◎ | △ | 정식 표준, 아직 Baseline 아님 |
| route-based modal | ◎ | ◎ | 공유 가능, 단순 모달엔 과설계 |

## 3. 결정

**용도에 따라 두 트랙으로 나눈다.**

### 트랙 A — 단순 확인/액션 오버레이 (평점 시트, 필터, 확인 다이얼로그)
**CloseWatcher 우선 + `history.pushState`/`popstate` fallback**을 공용 디렉티브/서비스(`OverlayDismiss`)로 추상화한다.

```javascript
// 의사코드: 공용 닫기 핸들러
function attachDismiss(overlayRef) {
  if ('CloseWatcher' in window) {
    const w = new CloseWatcher();
    w.onclose = () => overlayRef.close();
    return { requestClose: () => w.requestClose(), destroy: () => w.destroy() };
  }
  // fallback
  history.pushState('overlay-open', '');
  const onPop = () => overlayRef.close();
  addEventListener('popstate', onPop);
  return {
    requestClose: () => { if (history.state === 'overlay-open') history.back(); },
    destroy: () => removeEventListener('popstate', onPop),
  };
}
```

> **핵심 함정**: Close 버튼/Esc/배경 탭 등 직접 닫는 경로는 반드시 `requestClose()`를 거쳐 `history.back()`으로 가짜 엔트리를 정리한다. 안 그러면 다음 back이 헛돌아 페이지 이동이 한 번 어긋난다.

### 트랙 B — 공유/딥링크가 필요한 콘텐츠 (영화 상세)
**route-based modal**. 영화 상세는 canonical URL(`/movies/:id`)을 가지므로([ADR-0002](./ADR-0002-적응형-표현-전략.md)), 모달 열림 = 라우트 진입으로 본다. 뒤로가기로 닫힘, 앞으로가기로 재오픈, 새로고침 시 풀스크린 페이지. Angular Router로 자연 처리.

### 공통 정책
- 모든 오버레이는 **back으로 닫힘 + 명시적 Close(X) 버튼**을 둘 다 제공한다(grab handle만으론 불충분).
- back 닫기 자체를 막지 않는다. 저장 안 된 변경이 있으면 CloseWatcher `cancel`(또는 confirm) 로 한 번 되묻는다.

## 4. 주의사항

- **iOS Safari edge-swipe back**과 가로 스와이프 충돌: 인터랙티브 영역을 화면 가장자리에서 떨어뜨리고, dismiss는 vertical 스와이프 + back + 버튼으로 다중화한다.
- CloseWatcher는 가장 최근 watcher만 이벤트를 받으므로 중첩 오버레이는 LIFO로 닫힌다 — 트랙 A의 디렉티브가 스택을 일관되게 관리해야 한다.
- Angular Material Dialog의 `closeOnNavigation`은 라우터 네비에만 반응하므로 CDK 커스텀 시트에는 적용되지 않는다 — 트랙 A 디렉티브가 그 공백을 메운다.

## 5. 채택하지 않은 것

- **아무 처리 안 함**: 모바일 기대를 정면으로 위반.
- **모든 오버레이를 route-based로**: 단순 확인/액션까지 중첩 라우트를 만들면 과설계.
- **순수 `<dialog>`에 의존**: back-to-close가 공짜로 되지 않음.

## 6. 결과

- 👍 모바일에서 back 한 번으로 오버레이가 닫히는 자연스러운 동작.
- 👍 영화 상세는 공유·딥링크·새로고침까지 일관.
- 👎 CloseWatcher fallback과 히스토리 정리 로직을 디렉티브로 정확히 구현해야 함(테스트 필요).
