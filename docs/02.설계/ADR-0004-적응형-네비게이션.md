# ADR-0004 · 적응형 네비게이션

> 상태: ✅ 결정됨
> 일자: 2026-06-22
> 관련: [모바일 인터랙션 리서치 §3.1](../01.기획/모바일-인터랙션-리서치.md), [ADR-0002](./ADR-0002-적응형-표현-전략.md)

## 1. 맥락

주 내비게이션은 환경마다 기대가 다르다. 모바일은 엄지 도달성 때문에 **하단 탭바**를, 데스크톱은 **상단/사이드**를 기대한다. 또 햄버거 메뉴는 데스크톱에서 발견율을 크게 떨어뜨린다(NN/g: 발견율 20%+↓, 과업 시간 39%↑).

진입점: 홈 · 영화 · 검색 · (회원) 위시리스트/내 평점 · 로그인/로그아웃.

## 2. 검토한 대안

| 선택지 | 모바일 도달성 | 데스크톱 발견율 |
|--------|:---:|:---:|
| 양쪽 모두 햄버거 | △ | ✕ |
| 양쪽 모두 하단 탭바 | ◎ | △(데스크톱 부적합) |
| **모바일=하단 탭바 / 데스크톱=상단·사이드** ✅ | ◎ | ◎ |

## 3. 결정

**viewport 폭으로 내비게이션 형태를 분기한다.**

- **모바일**: 하단 탭바, 핵심 3~5개(홈·영화·검색·내 정보). 탭은 메뉴를 열지 말고 **목적지로 직접 이동**. 초과 항목은 "내 정보" 안으로.
- **데스크톱**: 상단 가로 네비 또는 좌측 사이드바, 항상 노출. **햄버거 지양.**
- 분기는 [ADR-0002](./ADR-0002-적응형-표현-전략.md)와 같은 브레이크포인트 토큰을 공유한다.
- 하단 탭바 패딩에 **`env(safe-area-inset-bottom)`** 적용(노치·홈 인디케이터 회피), 뷰포트에 `viewport-fit=cover`.
- 활성 탭 표시·포커스 순서·`aria-current`로 접근성 확보.
- 화면 전환에는 **View Transitions API**를 후보로 둔다(모바일의 페이지 전환 기대).

## 4. 주의사항

- 하단 탭바는 모바일 키보드가 올라올 때 가려질 수 있다 — 입력 화면에서는 숨기거나 `dvh`/VisualViewport로 보정([ADR-0008](./ADR-0008-모바일-입력-제스처-피드백.md)).
- 탭바가 콘텐츠 스크롤 영역을 잠식하지 않도록 본문 하단 패딩을 탭바 높이 + safe-area로 확보.

## 5. 채택하지 않은 것

- **데스크톱 햄버거**: 발견율·과업 시간 손해(NN/g).
- **모바일 상단 탭만**: 엄지 도달성 저하.

## 6. 결과

- 👍 각 환경에 맞는 도달성·발견율.
- 👍 safe-area로 노치 대응.
- 👎 두 내비 셸을 적응형으로 유지해야 함.
