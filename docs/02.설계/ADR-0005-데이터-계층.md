# ADR-0005 · 데이터 계층

> 상태: ✅ 결정됨
> 일자: 2026-06-22
> 관련: [데이터 저장 리서치](../01.기획/데이터-저장-리서치.md), [ADR-0006](./ADR-0006-인증.md)

## 1. 맥락

이 프로젝트는 **백엔드를 두지 않는다**([프로젝트 브리프 §8](../01.기획/프로젝트-브리프.md)). 영화·장르(읽기 전용 목 데이터)와 위시리스트·평점(런타임 생성)·설정·세션을 모두 브라우저에 영속화해야 한다. (자매 프로젝트 `angular-ssr-design`은 별도 Express + 프록시를 두지만, 그건 SSR 인증 포워딩을 시연하기 위함이다 — 우리 관심사가 아니다.)

## 2. 검토한 대안

| 데이터 | 후보 | 선택 |
|--------|------|------|
| 영화·위시리스트·평점 | localStorage / IndexedDB(raw) / Dexie | **Dexie(IndexedDB)** |
| 설정·세션 | localStorage / IndexedDB | **localStorage** |
| 임시 UI 상태 | 메모리 시그널 / sessionStorage | **메모리 시그널** |

## 3. 결정

- **영화·위시리스트·평점 → IndexedDB(Dexie.js).** 영화·장르는 정적 시드로 적재하고, 위시리스트·평점은 런타임에 쓴다. Dexie `liveQuery`를 Angular 시그널로 연결해 반응형 목록을 만든다.
- **설정·세션 → localStorage.** 소량 key-value, 동기 접근.
- **핵심 데이터는 `navigator.storage.persist()`로 영속을 요청**하고 `estimate()`로 사용량을 모니터링한다(IndexedDB는 best-effort라 eviction 가능).
- **저장소 접근은 Angular 서비스로 추상화**한다(`MovieRepository`, `WishlistRepository`, `RatingRepository`, `SettingsStore`). 화면·컴포넌트는 저장소 구현을 모르고 서비스 인터페이스만 안다 → 나중에 원격 백엔드가 생겨도 교체가 쉽다.

## 4. 주의사항

- 영속 권한(`persist()`)이 거부됐을 때의 UX(데이터가 사라질 수 있음 안내)를 정한다.
- Dexie 스키마 변경 시 버전 마이그레이션 전략.
- 다중 탭 동기화는 `BroadcastChannel` 또는 Dexie 변경 관찰로(필요 시).

## 5. 채택하지 않은 것

- **백엔드 서버 + 프록시**: 프로젝트 범위 밖(자매 프로젝트 주제).
- **RxDB / SQLite Wasm·OPFS**: 현재 규모에 과함. 서버 동기·대용량·복잡 SQL이 필요해지면 재검토.
- **위시리스트/평점을 localStorage에**: 구조화·증가·쿼리에 부적합.

## 6. 결과

- 👍 서버 없이 반응형(liveQuery→signal) 로컬 퍼스트 경험 + 오프라인 동작.
- 👍 저장소 추상화로 이후 동기화 도입 여지.
- 👎 영속 권한·마이그레이션·다중 탭 같은 클라이언트 저장소 함정을 직접 다뤄야 함.

## 7. 후속 작업

- [ ] Dexie DB 스키마(movies, genres, wishlist, ratings) + 시드
- [ ] Repository/Store 서비스와 `liveQuery`→signal 어댑터
- [ ] `persist()` 요청 + `estimate()` 모니터링
- [ ] `SettingsStore`(localStorage) 추상화
