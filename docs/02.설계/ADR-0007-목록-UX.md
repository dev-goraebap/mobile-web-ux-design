# ADR-0007 · 목록 UX

> 상태: ✅ 결정됨
> 일자: 2026-06-22
> 관련: [모바일 인터랙션 리서치 §3.2](../01.기획/모바일-인터랙션-리서치.md), [ADR-0005](./ADR-0005-데이터-계층.md)

## 1. 맥락

영화 목록(`/movies`, `/genre/:key`, `/search`)은 이 앱에서 가장 많이 보는 화면이다. 모바일 사용자는 **끊김 없는 스크롤**, **뒤로가기 시 스크롤 위치 복원**, **pull-to-refresh**를 기대한다. 데이터는 로컬(Dexie)이라 네트워크 페이징은 없지만, 큰 그리드의 DOM 수와 스크롤 경험을 관리해야 한다.

## 2. 검토한 대안

| 선택지 | 모바일 적합 | DOM 부담 | 접근성 |
|--------|:---:|:---:|:---:|
| 페이지네이션 | △ | 낮음 | ◎ |
| **무한 스크롤 + 가상 스크롤** ✅ | ◎ | 낮음 | △(보완 필요) |
| 전부 렌더 | ○ | 높음 | ○ |

## 3. 결정

- **무한 스크롤**(모바일 우선) + **CDK 가상 스크롤**로 화면에 보이는 만큼만 렌더해 DOM 수를 제어한다. 로컬 데이터를 청크로 끊어 점진 노출.
- **스크롤 위치 복원**: `history.scrollRestoration = 'manual'`로 두고, 목록↔상세 이동 후 뒤로가기 시 이전 위치를 직접 복원한다(SPA 비동기 렌더 때문에 자동 복원은 깨지기 쉬움).
- **pull-to-refresh**(모바일): 목록 상단에서 당겨 새로고침. `overscroll-behavior`로 스크롤 체이닝/네이티브 동작 제어.
- **접근성 보완**: 무한 스크롤만으로는 키보드/스크린리더 사용자가 불리하므로 **"더 보기" 버튼**을 병행하고, 새 항목 로드를 **`aria-live`**로 알리며 포커스를 관리한다.
- 그리드 **열 수는 viewport 폭에 적응**한다.
- 목록 항목의 액션(위시리스트 제거 등)은 모바일 **스와이프 + 항상 보이는 대체 버튼**([ADR-0008](./ADR-0008-모바일-입력-제스처-피드백.md)).

## 4. 주의사항

- 가상 스크롤 + 가변 높이 카드(포스터 비율 차이) 조합 시 높이 측정에 주의.
- 스크롤 복원은 데이터가 다시 채워진 뒤 복원해야 하므로 렌더 타이밍을 맞춘다.

## 5. 채택하지 않은 것

- **순수 페이지네이션**: 모바일에서 끊김. 단, 데스크톱 보조 수단으로는 열어둠.
- **전체 렌더**: 큰 그리드에서 DOM·메모리 부담.

## 6. 결과

- 👍 모바일에서 매끄러운 탐색 + 위치 복원 + pull-to-refresh.
- 👍 가상 스크롤로 DOM 경량.
- 👎 무한 스크롤 접근성 보완("더 보기"·aria-live)과 스크롤 복원 타이밍을 직접 구현.

## 7. 후속 작업

- [ ] CDK 가상 스크롤 그리드 + 청크 로딩
- [ ] `manual` 스크롤 복원(목록↔상세)
- [ ] pull-to-refresh 제스처
- [ ] "더 보기" 버튼 + `aria-live` + 포커스 관리
