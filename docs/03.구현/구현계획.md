# 구현 계획

> 설계 문서가 정의한 구조를 어떤 순서로 구현할지 정합니다. 원칙은 두 가지입니다. 시각 토대(디자인 토큰)를 먼저 깔고, 데모의 핵심인 적응형 Sheet를 일찍 만들어 접근을 검증합니다. 그 위에 데이터와 기능을 얹습니다.
> FSD 원칙에 따라 처음에는 페이지에 두고, 재사용이 실제로 생길 때 하위 레이어로 추출합니다.

## Phase 0 — 기반

모든 시각 작업이 여기에 의존하므로 가장 먼저 깔아 둡니다.

- [x] 경로 별칭 — `tsconfig.json`에 `@/*` → `src/*`
- [x] 디자인 토큰 — `src/styles.css`의 `@theme`에 Discord 토큰(색·간격·라운드·브레이크포인트) 정의
- [x] 대체 폰트 — Space Grotesk(디스플레이) + Inter(본문), Fontsource로 self-host
- [x] CDK 설치 — `@angular/cdk`, `overlay-prebuilt.css` import
- [x] `shared/ui`, `shared/lib` 골격 (첫 프리미티브를 만들 때 생성)

## Phase 1 — 디자인 시스템 프리미티브 + 적응형 Sheet

데모의 핵심입니다. Sheet 하나가 이후 모든 패턴의 참조 구현이 됩니다. 데이터는 아직 목업으로 둡니다.

- [x] Button — 가장 단순한 프리미티브로 토큰 파이프라인을 검증합니다 (DESIGN.md variants)
- [x] BreakpointObserver 래퍼(`shared/lib`) → signal로 노출
- [x] 적응형 Sheet — CDK Overlay + BreakpointObserver + FocusTrap. 모바일 바텀시트 ↔ 데스크톱 모달, `safe-area-inset-bottom`, 끌어서 닫기 제스처(GSAP Draggable + InertiaPlugin), 열기/닫기 애니메이션

## Phase 2 — 데이터 레이어

UI가 검증되면 영속화를 붙입니다.

- [x] 저장소 추상화 — Dexie DB(`shared/api/db.ts`) + `TodoStore`(CRUD), localStorage 추상화는 `shared/lib/local-storage-signal.ts`로 제공(구체 설정 서비스는 Phase 4)
- [x] `navigator.storage.persist()` 요청 — `shared/api/storage.ts`, `provideAppInitializer`로 시작 시 호출
- [x] Todo 모델 — 할 일 타입 + Dexie `liveQuery` → signal(`shared/lib/live-query.ts` 브리지)

## Phase 3 — 첫 기능 수직 슬라이스: 할 일

프리미티브와 저장소를 실제 기능으로 엮는 첫 사례입니다.

- [x] 홈 페이지 — 할 일 목록 (CDK 가상 스크롤) + FAB, `pages/home`로 라우팅(지연 로딩)
- [x] ListItem 프리미티브 (스와이프 가능, GSAP Draggable x)
- [x] add-todo — Sheet의 첫 실사용처(`pages/home/ui/add-todo-sheet`)
- [x] toggle-todo — 스와이프/체크박스 완료 + 되돌리기 Snackbar(삭제도 되돌리기 지원)
- [ ] 재사용이 확인되면 `entities/todo`로 추출 — 아직 단일 소비자라 `shared/api`에 둠(2번째 소비자 등장 시 추출)

## Phase 4 — 나머지 기능

- [x] 적응형 Nav(`widgets/app-nav`) — 하단 탭(모바일) ↔ 사이드 레일(laptop+), 활성 탭 primary
- [x] 설정 — 테마 전환(다크 기본 ↔ 라이트, `[data-theme=light]` 토큰 스왑, `ThemeService`로 localStorage 영속). 라이트 팔레트 = Discord 다크의 폴라리티 플립으로 확정
- [ ] 알림 — 리마인더, 권한 요청 흐름, 할일→알림 OHS 흐름
- [ ] 사용자 — 인증 (방식 확정 후 — 미정)

## Phase 5 — PWA 셸

- [x] `ng add @angular/pwa`, manifest(한국어 이름·테마색·standalone), service worker, 오프라인 캐싱(ngsw 기본 assetGroups), 설치 흐름(`PwaInstallService` + 설정의 "앱 설치")
- [x] `viewport-fit=cover` — 노치 기기에서 safe-area inset 실측 활성화
- [x] 번들 다이어트 — 라우트 지연 로딩으로 GSAP이 home 청크로 분리됨. 초기 번들이 다시 500kB 미만이라 예산을 500kB로 복원

## 진행 중 확정할 결정

- 라이트 테마 팔레트 — Phase 4 설정 시
- 인증 방식과 데이터 시드·목업 전략 — Phase 3~4 사이
