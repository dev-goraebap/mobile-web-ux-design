# 리서치 — Angular 반응형 UX 기술 스택

> 프로젝트를 시작하기 전에 조사한 내용을 정리합니다. 확정된 결정은 [프로젝트 브리프](%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8-%EB%B8%8C%EB%A6%AC%ED%94%84.md)에 두고, 이 문서에는 그 결정에 이르기까지 검토한 선택지와 근거를 담습니다.
> 이 문서는 **기술 스택·라이브러리 선택**을 다룹니다. 데스크톱/모바일 인터랙션 기대 차이, 적응형 모달/페이지, 뒤로가기 처리 등 **UX 패턴 리서치**는 [모바일 인터랙션 리서치](%EB%AA%A8%EB%B0%94%EC%9D%BC-%EC%9D%B8%ED%84%B0%EB%9E%99%EC%85%98-%EB%A6%AC%EC%84%9C%EC%B9%98.md)를 참고하세요.

## 1. Angular 22에서 AI의 도움

Angular는 22 버전 기준으로 AI 보조 개발을 세 갈래로 지원합니다. 각각 역할이 다르며, 함께 쓸 때 보완됩니다.

### AI 규칙 파일 (정적 지식)

Angular는 공식 모범 사례를 담은 `best-practices.md`를 배포합니다. 이 내용을 AI 도구의 시스템 지침으로 넣으면, 에이전트가 NgModules·`*ngIf`·생성자 주입 같은 구식 패턴 대신 standalone·signals·새 control flow를 사용합니다. `ng new`는 이 파일을 Claude Code용 `.claude/CLAUDE.md`로 자동 심어 줍니다.

- `angular.dev/llms.txt` — 주요 리소스 링크 색인
- `angular.dev/assets/context/llms-full.txt` — 컴파일된 전체 컨텍스트

### MCP 서버 (능동적 조회)

`ng mcp` 명령으로 Angular CLI MCP 서버를 띄우면, 에이전트가 정적 지식을 넘어 **실시간 도구**를 호출할 수 있습니다.

| 도구 | 역할 |
|------|------|
| `search_documentation` | angular.dev 공식 문서 검색 |
| `get_best_practices` | 버전별 모범 사례 로드 |
| `find_examples` | 공식 예제 코드 검색 |
| `list_projects` | `angular.json` 프로젝트 조회 |
| `onpush_zoneless_migration` | OnPush·zoneless 마이그레이션 계획 |

`--read-only`(조회 전용)와 `--local-only`(인터넷 불필요 도구만) 플래그로 범위를 제한할 수 있습니다.

### Agent Skills (패턴 강화)

`npx skills add angular/skills`로 설치합니다. 에이전트 중립 포맷이라 Claude Code·Gemini CLI 등에서 공통으로 동작합니다.

- **angular-developer** — 컴포넌트·서비스 생성, 시그널·라우팅·폼·DI·접근성·테스팅 모범 사례
- **angular-new-app** — CLI로 새 앱 생성 + 모던 구조 설정

## 2. 반응형 웹에서 네이티브에 가까운 경험 만들기

웹이 앱과 달라 보이는 지점은 대개 정해져 있습니다. 화면 가장자리 처리, 스크롤의 물성, 손가락에 대한 반응, 전환의 매끄러움입니다.

여기에는 한 가지 판단 기준을 둡니다. **표현은 직접 소유하되, 복잡한 동작 로직은 검증된 라이브러리에 맡깁니다.** safe-area나 스크롤 스냅처럼 선언적이고 단순한 것은 직접 구현하는 편이 가볍고 명확합니다. 반면 고수준 제스처는 속도·방향 판정, 멀티터치, 관성, 스냅, 브라우저 기본 동작과의 충돌처럼 엣지 케이스가 많아, 직접 구현하면 버그가 잘 생깁니다. 이런 영역은 이미 검증된 라이브러리를 쓰는 편이 안전합니다.

### 직접 구현 — 단순하고 선언적인 웹 표준

- **화면 가장자리 (safe-area)** — 노치와 홈 인디케이터를 피하려면 `viewport-fit=cover`와 `env(safe-area-inset-*)`를 사용합니다. 하단 고정 바나 바텀시트의 패딩에 특히 중요합니다.
- **스크롤의 물성** — `scroll-snap`으로 카드·페이지가 맺히게 하고, `overscroll-behavior`로 당겨서 새로고침이나 스크롤 체이닝을 제어합니다. 모멘텀 스크롤은 iOS 기본 동작에 맡깁니다.
- **전환 애니메이션** — 60fps를 유지하려면 레이아웃을 다시 계산하게 만드는 속성 대신 `transform`과 `opacity`만 사용합니다. 이 둘은 GPU 합성으로 처리됩니다. 화면 간 전환에는 View Transitions API도 후보입니다.
- **적응형 분기** — 같은 콘텐츠(예: 영화 상세)를 모바일에서는 풀스크린 페이지/바텀시트로, 데스크톱에서는 모달로 보여주려면 화면 크기를 관찰합니다. CDK `BreakpointObserver`나 CSS 컨테이너 쿼리를 씁니다. 분기 전략 자체는 [ADR-0002](../02.설계/ADR-0002-%EC%A0%81%EC%9D%91%ED%98%95-%ED%91%9C%ED%98%84-%EC%A0%84%EB%9E%B5.md)에서 결정합니다.
- **터치 친화** — 터치 타깃은 최소 44~48px를 확보하고, `:active` 상태로 누름 피드백을 즉시 줍니다.
- **앱 같은 셸** — PWA로 설치하면 주소창 없는 전체 화면과 스플래시를 얻습니다.

### 검증된 라이브러리 — 복잡한 동작 로직

고수준 제스처(스와이프로 닫기, 끌어서 여닫는 시트, 핀치 줌)와 관성·속도 기반 드래그는 직접 구현 대신 라이브러리를 후보로 둡니다. 정렬·드래그처럼 CDK가 이미 검증된 구현을 제공하는 영역은 CDK를 우선합니다.

| 라이브러리 | 강점 | 비고 |
|------------|------|------|
| **GSAP (Draggable + InertiaPlugin)** | 관성·속도 기반 스냅. "끌어서 여닫는 바텀시트"에 강함 | 2025년부터 전 플러그인 무료 + 상업적 사용 가능 |
| **Interact.js** | 드래그·드롭·리사이즈·핀치/회전, 포인터 이벤트 추상화 | 성숙·안정 |
| **@use-gesture** | 제스처를 정규화한 바닐라 코어. 스프링 애니메이션과 조합 | React 생태계 출신, 바닐라 패키지 제공 |
| **TinyGesture · The Finger** | 초경량(수 kB). 스와이프·탭·핀치 등 기본 제스처만 필요할 때 | 의존성 적음 |
| **Hammer.js** | 한때 표준이었던 안정적 구현 | 2016년 이후 유지보수 정지 — 신규 채택은 신중히 |
| **CDK DragDrop** | 정렬 리스트, 끌어서 옮기기 | 이미 채택된 토대 |

바텀시트를 예로 들면, 위치·백드롭·포커스 트랩은 CDK `Overlay`가, 손가락을 따라 끌리고 속도에 따라 닫히거나 스냅되는 드래그 동작은 위 라이브러리 중 하나가 맡는 식으로 역할을 나눌 수 있습니다.

### 성능

- **긴 리스트** — 항목이 많으면 가상 스크롤(CDK `Scrolling`)로 화면에 보이는 만큼만 렌더링합니다.

## 3. Angular에서 쓸 수 있는 UI 프레임워크·라이브러리

선택지는 "스타일을 얼마나 들고 오는가"로 나누면 정리됩니다.

### 풀 컴포넌트 키트 (동작 + 스타일)

바로 쓸 수 있는 완성된 컴포넌트를 제공합니다. 빠르지만, 그 디자인 언어를 함께 받아들이게 됩니다.

- **Angular Material** — 구글 공식, Material Design. 일관적이지만 Material 색채가 강합니다.
- **PrimeNG** — 컴포넌트 수가 방대하고 테마가 다양합니다.
- **NG-ZORRO** — Ant Design 기반. 데이터 밀도 높은 화면에 강합니다.
- **Taiga UI** — 대규모 컴포넌트 세트.
- **Nebular** — Eva Design 시스템 기반.

### 모바일·하이브리드

- **Ionic** — 모바일에 특화된 웹 컴포넌트 키트. Capacitor로 네이티브 패키징까지 이어집니다. 다만 컴포넌트를 Shadow DOM에 캡슐화하므로, 외부 Tailwind 클래스가 내부에 닿지 않습니다.

### 헤드리스 (동작만, 스타일 없음)

- **Angular CDK** — Overlay·DragDrop·A11y·Scrolling 등 동작과 접근성만 제공합니다. 마크업은 전부 light DOM에 남으므로 스타일을 자유롭게 입힐 수 있습니다. Material도 이 위에 지어졌습니다.
- **Spartan/ui** — shadcn 스타일의 Angular 구현. CDK와 Tailwind를 묶어 복붙 가능한 컴포넌트로 제공합니다. 디자인이 어느 정도 입혀져 있어 "완전 소유"와는 한 발 떨어지지만, 참고 구현으로 유용합니다.

### CSS·유틸리티 (프레임워크 무관)

- **Tailwind CSS** — 유틸리티 클래스로 디자인을 직접 조립합니다. 디자인 토큰과 결합하기 좋습니다.
- **DaisyUI · Flowbite** — Tailwind 위에 컴포넌트 스타일을 얹은 CSS 레이어.

## 4. 이 프로젝트에 어울리는 기술 스택

이 프로젝트의 목표는 두 가지였습니다. 모바일 경험을 우선하고, 디자인을 완전히 소유하는 것입니다. 이 기준으로 위 선택지를 거르면 결론이 좁혀집니다.

**채택: Angular CDK(헤드리스) + Tailwind CSS v4 + 직접 구현한 모바일 레이어.**

- 풀 컴포넌트 키트(Material·PrimeNG 등)는 디자인 언어가 함께 따라오므로, "디자인 소유" 목표와 맞지 않습니다.
- Ionic은 모바일에 강하지만 Shadow DOM이 Tailwind를 가로막습니다. 컴포넌트 내부까지 디자인을 통제하려는 의도와 충돌합니다.
- CDK는 동작과 접근성이라는 어려운 부분만 가져오고 마크업을 전부 light DOM에 남기므로, Tailwind가 모든 요소에 적용됩니다. 디자인 자유도와 검증된 접근성을 동시에 얻습니다.
- safe-area·스크롤 물성 같은 단순한 영역은 2절의 웹 표준 기법으로 직접 구현합니다. 반면 고수준 제스처처럼 복잡도가 높은 동작은 직접 만들지 않고 검증된 라이브러리(GSAP Draggable·InertiaPlugin, Interact.js, @use-gesture 등)를 후보로 둡니다. 최종 선정은 첫 바텀시트를 구현하며 후보를 좁힙니다.
- Spartan/ui는 같은 토대(CDK + Tailwind)를 쓰므로, 직접 만들기 전에 구현을 참고할 대상으로 둡니다.

확정된 스택과 범위는 [프로젝트 브리프](%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8-%EB%B8%8C%EB%A6%AC%ED%94%84.md)에 정리되어 있습니다.
