# 프로젝트 브리프 — 영화 카탈로그 (반응형 UX 데모)

> 작성일: 2026-06-22
> 범위: **도메인/기능/콘텐츠 기획**과 **확정 기술 스택**을 다룹니다. 화면별 표현 전략·인터랙션 결정은 [`docs/02.설계/`](../02.설계/)의 ADR에서 별도로 다룹니다.
> 검토 과정과 대안은 [리서치](%EB%A6%AC%EC%84%9C%EC%B9%98.md)·[모바일 인터랙션 리서치](%EB%AA%A8%EB%B0%94%EC%9D%BC-%EC%9D%B8%ED%84%B0%EB%9E%99%EC%85%98-%EB%A6%AC%EC%84%9C%EC%B9%98.md)를 참고하세요.

## 1. 무엇을 만드나

영화 카탈로그를 만듭니다. 다만 목적은 제품 자체가 아니라, 그 위에서 **반응형 웹 UX 전략을 구현하고 검증**하는 데 있습니다.

핵심 질문은 하나입니다 — **하나의 코드베이스로, 데스크톱 웹과 모바일 웹에서 각각 그 환경에 맞는(네이티브에 가까운) 경험을 어떻게 제공할 것인가?** 같은 화면이라도 사용자가 기대하는 동작은 환경마다 다릅니다. 데스크톱에서는 모달이 자연스럽지만 모바일의 작은 화면에서는 풀스크린이 편하고, 모바일에서는 뒤로가기 한 번으로 오버레이가 닫히길 기대합니다.

도메인으로 **영화 카탈로그**를 선택한 이유:

- **콘텐츠 탐색(목록·상세·그리드)** 과 **개인화(위시리스트·내 평점)** 가 자연스럽게 공존한다 → 반응형 UX 패턴(적응형 모달/페이지, 무한 스크롤, 공유, 바텀시트)을 한 도메인에 풍부하게 담을 수 있다.
- 영화 상세는 **"데스크톱=모달 / 모바일=페이지"** 적응형 표현의 교과서적 사례다(Instagram·Pinterest 사진 상세와 동일한 모델).
- 데이터 모델이 단순하다(영화 1개 엔티티 중심).
- 누구나 이해하는 친숙한 도메인이라 UX 데모의 본질에 집중할 수 있다.

> 데이터는 외부 API 없이 **자체 목(mock) 데이터**로 구성하고 브라우저에 영속화한다(실제 영화 서비스가 아님).
> 자매 프로젝트 `angular-ssr-design`는 같은 영화 도메인을 **SSR 렌더링 전략**으로 다룬다. 이 프로젝트는 같은 도메인을 **CSR + 반응형 UX 전략**으로 다뤄, 둘을 비교할 수 있게 한다.

## 2. 핵심 사용자 & 시나리오

### 사용자 유형

| 유형 | 설명 | 인증 |
|------|------|------|
| **방문자(Guest)** | 로그인 없이 둘러보는 사용자 | 불필요 |
| **회원(Member)** | 로그인 후 위시리스트/평점을 관리하는 사용자 | 필요 |

> 회원 가입 절차는 시연 단순화를 위해 생략하고, **데모 계정 로그인**만 제공한다(아이디/비밀번호 1쌍).

### 대표 시나리오

1. **둘러보기**: 방문자가 홈에 들어와 추천/최신 영화를 보고, 목록에서 마음에 드는 영화 상세로 들어간다.
2. **탐색**: 장르로 필터링하거나 제목으로 검색해 원하는 영화를 찾는다.
3. **찜하기**: 회원이 영화를 위시리스트에 담고, 나중에 위시리스트에서 다시 본다.
4. **평가하기**: 회원이 본 영화에 별점을 남기고, "내 평점" 목록에서 관리한다.
5. **공유하기**: 마음에 드는 영화 상세 링크를 친구에게 공유한다(모바일=네이티브 공유 시트, 데스크톱=링크 복사).

## 3. 도메인 모델

### 3.1 Movie (영화) — 핵심 엔티티

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 식별자 (URL slug 겸용 가능) |
| `title` | string | 제목 |
| `originalTitle` | string? | 원제 |
| `year` | number | 개봉 연도 |
| `genres` | string[] | 장르 키 목록 (예: `["action", "sci-fi"]`) |
| `runtime` | number | 상영 시간(분) |
| `synopsis` | string | 줄거리 |
| `director` | string | 감독 |
| `cast` | string[] | 주요 출연진 |
| `posterColor` | string | 포스터 대체용 색상(목 데이터라 실제 이미지 대신 컬러 블록 사용) |
| `posterUrl` | string? | (선택) 포스터 이미지 경로 |
| `ratingAverage` | number | 평균 평점 (0~5) |
| `ratingCount` | number | 평점 참여 수 |
| `featured` | boolean | 홈 추천 노출 여부 |

### 3.2 Genre (장르)

| 필드 | 타입 | 설명 |
|------|------|------|
| `key` | string | 식별자 (URL용, 예: `sci-fi`) |
| `label` | string | 표시명 (예: `SF`) |

초기 장르 셋(예): 액션, 드라마, SF, 코미디, 스릴러, 애니메이션, 로맨스, 공포.

### 3.3 User (사용자/세션)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 사용자 식별자 |
| `name` | string | 표시 이름 |

세션은 로그인 시 발급되는 토큰으로 식별한다(구체적 인증 방식은 [ADR-0006](../02.설계/ADR-0006-%EC%9D%B8%EC%A6%9D.md)에서 결정).

### 3.4 WishlistItem (위시리스트 항목)

| 필드 | 타입 | 설명 |
|------|------|------|
| `userId` | string | 소유자 |
| `movieId` | string | 대상 영화 |
| `addedAt` | string(ISO) | 담은 시각 |

### 3.5 Rating (내 평점)

| 필드 | 타입 | 설명 |
|------|------|------|
| `userId` | string | 평가자 |
| `movieId` | string | 대상 영화 |
| `score` | number | 별점 (1~5) |
| `ratedAt` | string(ISO) | 평가 시각 |

### 관계 요약

```
User 1 ── N WishlistItem N ── 1 Movie
User 1 ── N Rating       N ── 1 Movie
Movie N ── N Genre
```

## 4. 화면(페이지) 구성

> 각 화면의 **적응형 표현(모달/페이지/시트)** 과 인터랙션은 [ADR-0002](../02.설계/ADR-0002-%EC%A0%81%EC%9D%91%ED%98%95-%ED%91%9C%ED%98%84-%EC%A0%84%EB%9E%B5.md)·[ADR-0004](../02.설계/ADR-0004-%EC%A0%81%EC%9D%91%ED%98%95-%EB%84%A4%EB%B9%84%EA%B2%8C%EC%9D%B4%EC%85%98.md)에서 결정한다.

| # | 경로 | 화면 | 인증 | 핵심 내용 | 적응형 표현 |
|---|------|------|------|-----------|-------------|
| 1 | `/` | **홈/랜딩** | 공개 | 추천(featured) 영화, 장르 바로가기 | 동일 |
| 2 | `/movies` | **영화 목록** | 공개 | 전체 영화 그리드, 정렬(연도/평점), 무한 스크롤 | 그리드 열 수 적응 |
| 3 | `/movies/:id` | **영화 상세** | 공개 | 줄거리·감독·출연·평점, 위시리스트 담기, 별점 남기기, 공유 | **데스크톱=모달 / 모바일=페이지** |
| 4 | `/genre/:key` | **장르별 목록** | 공개 | 특정 장르의 영화 목록 | 그리드 적응 |
| 5 | `/search` | **검색 결과** | 공개 | 제목 검색 결과(쿼리 파라미터 `q`) | 검색 진입: 데스크톱=드롭다운 / 모바일=풀스크린 |
| 6 | `/wishlist` | **내 위시리스트** | 회원 | 담은 영화 목록, 제거(스와이프) | 동일 |
| 7 | `/me/ratings` | **내 평점** | 회원 | 내가 남긴 별점 목록, 수정/삭제 | 평점 수정: 데스크톱=모달 / 모바일=바텀시트 |
| 8 | `/login` | **로그인** | 공개 | 데모 계정 로그인, 로그인 후 원래 페이지로 복귀 | 동일 |

### 글로벌 내비게이션

- 핵심 진입점: **홈 · 영화 · 검색 · (회원) 위시리스트/내 평점 · 로그인/로그아웃**
- 모바일 우선 설계 — 형태(하단 탭바 vs 사이드/상단)는 [ADR-0004](../02.설계/ADR-0004-%EC%A0%81%EC%9D%91%ED%98%95-%EB%84%A4%EB%B9%84%EA%B2%8C%EC%9D%B4%EC%85%98.md)에서 결정.

## 6. 기능 요구사항

### 공개(비회원 포함)
- [ ] 홈에서 추천 영화·장르 진입점 노출
- [ ] 영화 목록 조회 (정렬: 최신순/평점순, 무한 스크롤)
- [ ] 영화 상세 조회 (줄거리, 메타데이터, 평균 평점) — 적응형 모달/페이지
- [ ] 장르별 필터링
- [ ] 제목 검색
- [ ] 영화 공유 (네이티브 공유 시트 / 링크 복사)

### 회원 전용
- [ ] 데모 계정 로그인 / 로그아웃
- [ ] 위시리스트 담기 / 빼기 / 목록 조회
- [ ] 별점 남기기 / 수정 / 삭제 / 내 평점 목록 조회
- [ ] 비로그인 상태에서 회원 기능 시도 시 로그인으로 유도 후 복귀

### 비기능(도메인 관점)
- [ ] 모바일 우선(Mobile-first) 경험 + 데스크톱 적응
- [ ] 오버레이는 뒤로가기로 닫힘 + 명시적 닫기 버튼 ([ADR-0003](../02.설계/ADR-0003-%EB%92%A4%EB%A1%9C%EA%B0%80%EA%B8%B0-%EC%98%A4%EB%B2%84%EB%A0%88%EC%9D%B4-%EB%8B%AB%EA%B8%B0.md))
- [ ] 접근성: WCAG AA 준수(포커스 관리, 색 대비, ARIA) — 프로젝트 공통 규칙

## 7. 콘텐츠/데이터 규모(시연 기준)

- 영화: **약 15~20편** (장르가 골고루 분포하도록 구성)
- 장르: **6~8종**
- 데모 계정: **1개** (예: `demo` / `demo1234`)
- 위시리스트/평점: 로그인 후 런타임에 생성(초기엔 비어 있음)

---

## 8. 범위 밖 (Non-Goals)

- **SSR · 라우트별 혼합 렌더링** — 이 프로젝트는 CSR + 반응형 UX에 집중한다. SSR은 그 자체로 깊은 주제이므로 자매 프로젝트(`angular-ssr-design`)에서 다룬다.
- **백엔드(서버)** — 서버는 두지 않는다. 데이터는 브라우저에 영속화한다([데이터 저장 리서치](%EB%8D%B0%EC%9D%B4%ED%84%B0-%EC%A0%80%EC%9E%A5-%EB%A6%AC%EC%84%9C%EC%B9%98.md)). 저장소 접근은 서비스로 추상화해 이후 도입 여지만 남긴다.
- 실제 회원가입/비밀번호 재설정/소셜 로그인
- 결제·예매·스트리밍 등 실제 서비스 기능
- 외부 영화 API 연동(목 데이터로 대체)
- 관리자(영화 등록/수정) 화면
- 댓글·리뷰 텍스트(별점만 다룸)
- 다국어(i18n)
